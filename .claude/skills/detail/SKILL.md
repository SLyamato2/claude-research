# /detail - 番号指定で詳細解説スキル

検索結果の番号を指定して、その情報の詳細解説を表示する。

## 使い方
```
/detail <番号>
```

## 実行手順

1. **結果の特定**: 直前の検索結果から指定された番号の項目を特定する（`results/latest.md` がなければ会話内の検索結果を参照）

2. **情報取得**: WebFetchツールで該当URLの内容を取得する

3. **詳細解説の出力**: 以下のフォーマットで日本語で出力する
   ```
   ## #<番号> の詳細解説

   **タイトル:** <タイトル>
   **URL:** <URL>
   **ソース:** <YouTube/X/Dev.to/Hashnode/Hacker News/Zenn/note>
   **カテゴリ:** <公式・アップデート/ユーザー発見・活用法/トレンド・バズ>

   ### 内容の詳細
   （WebFetchで取得した情報を元に、内容を詳しく日本語で解説）

   ### こんな場面で使える
   （読者が「自分ごと」としてイメージできるよう、具体的な利用シーンを3つ以上挙げる）
   - **場面1:** どんな状況の人が → 何をすると → どう良くなるか
   - **場面2:** ...
   - **場面3:** ...
   ※ 抽象的な説明ではなく「○○を開発しているとき」「○○で困っているとき」のように
     読者の日常業務に紐づけて書くこと

   ### 実践ステップ
   （今すぐ試せる具体的な手順。コマンド例・設定例・コードスニペットを含める）
   1. ステップ1: ...
   2. ステップ2: ...
   3. ステップ3: ...

   ### ポイント・注意点
   （重要な注意点、前提条件、制限事項など）
   ```

## 注意事項
- 日本語で出力すること
- `results/latest.md` が存在しない場合は、会話内の直前の検索結果を参照する
- WebFetchで取得できない場合は、WebSearchで補足情報を検索してから解説する
- **「こんな場面で使える」セクションが最重要**。読者が自分の業務のどこで活かせるか具体的にイメージできるように書く
- 具体例やコマンド例は可能な限り含める
